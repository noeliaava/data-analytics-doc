Database Management System

Una DBA (Database Administrator) es la profesional a cargo de todas las funciones involucradas en el desarrollo, almacenamiento, seguridad y recuperación de bases de datos. Trabaja con software de sistemas de gestión de bases de datos y determina formas de organizar y almacenar datos.

Un DBA se centra en los aspectos físicos y técnicos de la gestión de bases de datos que mantienen los sistemas funcionando sin problemas, de forma eficiente y sin errores. Se encarga de asegurar de que los datos estén seguros y evitar la pérdida de datos. También trata con la actualización de permisos y la instalación de actualizaciones y parches para corregir errores del programa de software.

Los DBA pueden ser responsables de varias aplicaciones en diferentes sistemas, fusionando bases de datos antiguas en nuevas bases de datos y realizando y probando modificaciones en las estructuras de la base de datos cuando sea necesario.

Una DBA trabaja con DBMS (DataBase Management System). El software DBMS funciona como una interfaz entre el usuario y la base de datos, administrando simultáneamente los datos, el motor de la base de datos y el esquema de la base de datos para facilitar la organización y manipulación de los datos.

Un sistema de administración de base de datos (DBMS) funciona mediante el uso de comandos del sistema, primero recibe instrucciones de un DBA en DBMS y luego instruye al sistema en consecuencia, ya sea para recuperar datos, modificar datos o cargar datos existentes del sistema. Los ejemplos más populares de DBMS incluyen sistemas de administración de bases de datos basados ​​en la nube, sistemas de administración de bases de datos en memoria (IMDBMS), sistemas de administración de bases de datos en columnas (CDBMS) y NoSQL en DBMS.

Garantiza

Integridad de datos

La primera responsabilidad es mantener la integridad de los datos de la empresa, porque incluso si una base de datos es rápida y accesible, es inútil si los datos a los que se accede están distorsionados. La integridad de los datos debe gestionarse con precisión porque se trata de proteger los datos del uso no autorizado. Un DBA administra la relación entre los datos para mantener la coherencia de los mismos.

La estructura interna de una base de datos consta de datos almacenados en tablas. El acceso a los datos se puede optimizar mediante el uso de índices, que se leen como el índice de un libro. Los DBA ayudan a garantizar que la base de datos esté optimizada para aprovechar el hardware físico o virtual en el que está alojada. Esto significa que es necesario poder dividir los datos en diferentes discos, configurar la base de datos para utilizar mejor la memoria o la RAM y optimizar el tráfico de red de las bases de datos remotas. Algunas bases de datos incluso se dividen entre varios servidores en las llamadas "configuraciones distribuidas". Algunas bases de datos pueden volverse tan complejas que necesitan un DBA que se centre únicamente en la mejora del rendimiento.

Garantiza

Escenario de restauración de base de datos.

Se concentra en monitorear y mantener un alto nivel de rendimiento del sistema. Los administradores de base de datos son responsables de realizar copias de seguridad periódicas del sistema para que no se pierdan datos durante un corte de energía u otro tipo de desastre. También trabajan para asegurarse de que los datos que se capturan sean confiables, estén libres de errores y estén disponibles en cualquier momento.

El DBA tiene que decidir qué datos se deben utilizar y qué tipo de datos son precisos para la empresa, de acuerto a las reglas y las necesidades del negocio. Por lo tanto, siempre realiza controles de validación en los datos para que sean más precisos y consistentes.

Da soporte

Esquema de asignación de permisos

La DBA crea subesquemas para decidir sobre la accesibilidad de la base de datos. De acuerdo a las necesidades del negocio, crea y administra los usuarios de la base de datos y también qué datos utilizará cada usuario. Ningún usuario tiene que poder acceder a toda la base de datos sin el permiso del DBA.

La capacidad de otorgar selectivamente el derecho a manipular y acceder a la información a través de privilegios es integral para garantizar la seguridad de la base de datos. Los DBA son usuarios con autoridad para especificar nuevos usuarios, especificar roles de usuarios y otorgar privilegios generales a los usuarios. También pueden revocar los privilegios y eliminar usuarios.

Si algún usuario necesita ayuda en cualquier momento, es una función del DBA ayudarlo. El DBA brinda soporte completo a los usuarios que son nuevos en la base de datos.
